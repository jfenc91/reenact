Why reenact?

Screenshots go stale. Screen recordings take manual effort. reenact generates demo videos automatically from code.

AI agent friendly

YAML scripts are easy for AI agents to generate. Let Claude, GPT, or any LLM create demo recordings from diffs, PRs, or feature descriptions.

CI/CD ready

Run in GitHub Actions, GitLab CI, or any pipeline. Auto-generate demo videos on every PR and post them as comments.

Real browser, real video

Playwright drives actual Chromium — WebGL, SPAs, auth flows all work. Output is WebM or MP4, ready to share.

Looks human

Bezier mouse curves, variable-speed typing, smooth scrolling. Recordings look like a person used the app, not a bot.

Simple YAML scripts

Describe what to click, type, and scroll in plain YAML. Version-controlled, diffable, and easy for humans and AI to read and write.

Ship to PRs

Drop a .reenact/demo.yaml in your repo. Every PR gets a video showing the change — automatically.

How it works

1

Define or generate a script

Write YAML by hand, or let an AI agent generate it from a PR diff, feature spec, or bug report.

2

reenact records it

Headless Chromium executes every step with realistic mouse movement and typing while recording video.

3

Share the video

Post to PRs, embed in docs, attach to tickets. WebM or MP4 — works everywhere.

Getting started

Requirements

Node.js 18+

Install

$ npm install -g @jfenc91/reenact

Your first recording

Create demo.yaml:

url: https://example.com
steps:
  - wait: 1s
  - click: "h1"
  - scroll: down

Run it:

$ reenact demo.yaml -o demo.webm

Examples

Copy any of these, swap the URLs and selectors for your app, and run.

Login flow

Form interaction
View YAML script 
# Record a realistic login interaction
name: Login Flow
url: https://the-internet.herokuapp.com/login
viewport:
  width: 1280
  height: 720

steps:
  - wait: 1s

  # Type credentials naturally
  - type:
      selector: "#username"
      text: "tomsmith"
      wpm: 65

  - wait: 300ms

  - type:
      selector: "#password"
      text: "SuperSecretPassword!"
      wpm: 55

  - wait: 500ms
  - click: "button[type='submit']"
  - wait: 2s
  - scroll: down

Action reference

Every step you can use in a script.

click

- click: "#btn"

Smooth mouse move to element, then click at a random point within its bounds.

type

- type: selector: "#input" text: "hello" wpm: 70

Type character by character at natural speed. wpm controls pace (default 70).

hover

- hover: ".menu-item"

Move the cursor over an element without clicking. Great for tooltips and menus.

scroll

- scroll: down - scroll: direction: up amount: 500

Smooth incremental scroll. Direction is up or down, amount in pixels (default 300).

wait

- wait: 2s - wait: "#loaded"

Pause for a duration (ms, s, m) or wait until a selector appears.

wait_for

- wait_for: selector: ".modal" state: visible timeout: 10s

Wait for an element to reach a state: visible, attached, hidden, detached.

navigate / goto

- navigate: https://example.com

Navigate to a new URL. Waits for DOM content to load.

key / press

- key: Enter - key: Control+A

Press a keyboard key or combination.

fill

- fill: selector: "#bio" text: "long content..."

Instantly fill a field (non-humanized). Use for long text where typing would be slow.

select

- select: selector: "#country" value: "US"

Choose a value from a <select> dropdown.

clear

- clear: "#search"

Clear the contents of an input field.

screenshot

- screenshot: output.png

Capture a screenshot mid-recording. Useful for debugging or docs.

back / forward / reload

- back - forward - reload

Browser history navigation and page reload.

move_to

- move_to: x: 500 y: 300 duration: 600

Move cursor to exact coordinates with smooth bezier motion.

Demo videos on autopilot.

# 1. Create a script
$ cat > demo.yaml <<EOF
url: https://your-app.com
steps:
  - wait: 1s
  - click: ".get-started"
  - wait: 2s
EOF

# 2. Record it
$ reenact demo.yaml -o demo.webm

# 3. That's it. You have a video.
